    1. Adds `"use client"` directive at top
    2. Imports `gsap` from `"gsap"`, `ScrollTrigger` from `"gsap/ScrollTrigger"`
    3. Calls `gsap.registerPlugin(ScrollTrigger)` once at module level
    4. Re-exports `gsap` and `ScrollTrigger` as named exports so all consumers import from `@/lib/gsap` not directly from `"gsap"`
    5. Does NOT import `@gsap/react` here — that is imported per-component alongside this module

    **lib/useReducedMotion.ts** — Per D-08, create a shared hook:
    1. Adds `"use client"` directive
    2. Uses `useState` + `useEffect` to read `window.matchMedia('(prefers-reduced-motion: reduce)')` on mount
    3. Listens for changes via `addEventListener('change', ...)` so toggling OS setting mid-session takes effect
    4. Returns `boolean` — `true` means user prefers reduced motion
    5. SSR-safe: defaults to `false` during SSR (no window), updates on mount
    6. This hook is consumed by: GSAP components (to skip timeline), Framer Motion components (to disable transitions), Three.js wrappers (to show static fallback)

    Do NOT use Framer Motion's built-in `useReducedMotion()` — we need a framework-agnostic hook that works with all 3 systems identically.
